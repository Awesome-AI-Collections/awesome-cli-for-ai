# Logo Generator Skill

## 一句话总结

一个面向 Claude Code 等 agent 宿主的品牌设计 skill，可让 AI 直接生成多套 SVG logo 方案和展示图，把“想法到初版品牌视觉”压缩成一条终端工作流。

## 快速判断

- 如果你想让 agent 在终端里直接产出 logo 概念和展示图，它很值得优先看
- 如果你需要 Figma 级手工微调、完整品牌规范和高级设计审美把控，先别把它当成最终方案
- 它更像“品牌设计自动化 skill”，不是完整设计平台

## 你会怎么用它

- 接进 AI 开发流：把 skill 安装进 Claude Code 或其他兼容宿主，让 agent 根据产品描述直接生成 logo 方案
- 接进日常工作流：让产品、设计或独立开发者快速产出多版视觉方向，再挑一版继续细化
- 最小落地方式：先给它一个清晰的产品定位和风格要求，跑一次 3-6 个 logo 变体

## 它解决什么问题

- 很多人能写产品说明，但很难快速把品牌概念变成多套可比较的 logo 初稿
- 用通用模型直接生图，往往缺少 SVG、展示图和方案整理这类完整交付
- 团队在品牌探索早期经常只是想“先出几版看看”，但手工来回切换工具很耗时间

## 适合谁

- 独立开发者和小团队
- 想让 agent 帮忙跑品牌探索的产品经理和设计师
- 需要快速验证命名、定位和视觉方向是否匹配的人

## 核心能力

- Logo 方案生成：围绕同一品牌 brief 输出多套 SVG logo 方向
- 展示图产出：不只给图形本体，也补展示与预览效果
- Agent 可调用：以 skill 形式接入宿主，适合自然语言驱动
- 视觉工作流打包：把提示、生成、变体整理和输出串成一套终端流程

## 能力边界

- 它强在“快速出多套方向”，不是最终品牌系统交付工具
- 结果质量仍受品牌 brief 清晰度和模型能力影响
- 如果你需要严格控制字距、栅格、印刷规范和长期视觉系统，仍要设计师继续打磨

## 集成方式

- 作为单独工具：把 skill 装进宿主后，直接用自然语言生成 logo 概念
- 作为 AI 工作流组件：适合放在“品牌命名后、正式设计前”的概念探索阶段
- 作为团队流程节点：可让产品或增长团队先做视觉方向收敛，再交给设计继续精修

## 上手建议

- 第一步先把产品定位、受众、关键词和不想要的风格写清楚
- 最值得先试的场景是：同一产品跑 2-3 组不同风格 brief，对比哪种方向更对味
- 如果你只是想验证方向，先轻量试用，不要一开始就要求它完成最终品牌定稿

## 选型建议

- 如果你的主需求是“让 agent 直接帮你出 logo 初稿”，它很合适
- 如果你的主需求是“完整品牌系统设计”，更适合把它当作前置探索层
- 如果你看重的是 skill 化交付和终端内调用，而不是网页设计器，这个项目定位很清楚

## 典型使用场景

- 给新产品或 side project 快速生成第一版 logo 候选
- 用 agent 批量测试不同品牌关键词和视觉方向
- 在终端工作流里把命名、定位和视觉探索串起来

## 为什么值得关注

- 它不是普通 prompt 仓库，而是把 logo 生成真正封装成 skill
- 对喜欢在 Claude Code、Codex 里把创意工作流自动化的人非常顺手
- 对早期项目来说，先拿到多套可比较方向，比空想品牌风格更高效
