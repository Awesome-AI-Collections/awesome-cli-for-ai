# Show Me The Money

## 一句话总结

一个面向 Claude Code 和 Codex CLI 的商业变现技能包，用 `/money` 这类命令把“想法、产品、客户、收入”串成可执行的终端工作流。

## 快速判断

- 如果你想让 agent 不只会写代码，还能帮你推进 offer、获客和变现动作，它值得优先看
- 如果你需要的是真实支付、CRM、广告投放或财务系统本身，不要把它当成执行系统
- 它更像“终端里的商业增长操作层”，不是独立 SaaS 平台

## 你会怎么用它

- 接进 AI 开发流：把技能包装进 Claude Code 或 Codex CLI，让 agent 按 `/money` 路径推进商业问题
- 接进日常工作流：在做产品验证、定价、包装方案和增长动作时，用它做结构化推进
- 最小落地方式：拿一个现成想法，先跑一轮“目标客户 + offer + 首个收入动作”的 workflow

## 它解决什么问题

- 很多 agent 会写代码，但不会系统化地帮你想“怎么赚钱”
- 产品从点子到变现之间常常断层，缺少可复用的业务工作流模板
- 创业者和增长团队想把商业分析沉淀成固定步骤，而不是每次从空白开始问模型

## 适合谁

- 独立开发者和 AI 创业者
- 做产品验证、增长和商业化探索的人
- 想在终端里把业务工作流标准化的 Claude Code / Codex 用户

## 核心能力

- Slash-command 入口：通过 `/money` 等命令把商业问题导入固定流程
- 商业工作流模板：围绕 idea、offer、客户、获客和收入给出成套步骤
- CLI 宿主兼容：明确面向 Claude Code 和 Codex CLI 这类终端 agent
- 业务动作串联：让 agent 从“讨论想法”进一步走到“生成具体下一步”

## 能力边界

- 它强在“把变现思路流程化”，不是支付或 CRM 执行工具
- 最终商业判断仍取决于真实市场反馈，不能把 skill 输出当成事实本身
- 如果没有基本产品信息、目标客户和业务约束，结果会偏空泛

## 集成方式

- 作为单独工具：把技能包装进宿主后，直接通过 slash command 调用
- 作为 AI 工作流组件：适合放在“产品验证 / 定价 / GTM 草案 / 收入实验”这一层
- 作为团队流程节点：可用于产品、增长和创始人会议前的方案准备与结构化思考

## 上手建议

- 第一步先准备好你的产品一句话描述、目标客户和当前卡点
- 最值得先试的场景是：拿一个已有 side project，要求它给出第一批变现动作而不是空泛建议
- 如果你还没有真实用户和反馈，先把它当思路加速器，不要直接当成最终商业决策

## 选型建议

- 如果你的主需求是“让 CLI agent 帮我推进变现思考”，它很合适
- 如果你的主需求是“真正连接支付、CRM、广告平台”，还需要搭配别的工具
- 如果你看重的是终端工作流和可复用商业模板，而不是网页 SaaS，这个项目定位很清楚

## 典型使用场景

- 为新项目快速梳理目标客户、offer 和首个收入实验
- 让 Claude Code / Codex 在写产品的同时也给出商业化下一步
- 把创业和增长思考沉淀成固定 slash command 工作流

## 为什么值得关注

- 它明确补的是很多编码 agent 的盲区：商业化和收入动作
- GitHub 描述直接点明“从 idea 到 product 到 customer 到 money”，定位很清楚
- 对终端重度用户来说，把增长工作流塞回 CLI 里会非常顺手
