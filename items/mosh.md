# Mosh

## 一句话总结

一个面向不稳定网络环境的移动终端 shell，用更稳的远程会话、漫游和本地回显体验替代传统交互式 SSH。

## 它解决什么问题

- SSH 在弱网、切网、跨国网络和高延迟环境下体验经常很差
- 笔记本休眠、网络闪断或 IP 漫游时，远程会话容易中断
- 远程输入必须等服务器回包，终端交互迟钝明显

## 适合谁

- 经常远程连服务器、开发机和云主机的终端用户
- 需要在 Wi-Fi、蜂窝网络和移动场景下保持会话稳定的人
- 对终端交互延迟和鲁棒性非常敏感的开发者与运维人员

## 核心能力

- 网络漫游：切换网络后尽量保持会话不断开
- 弱网恢复：短时掉线、休眠和恢复后可继续会话
- 本地回显：输入和行编辑更即时，弱网下主观延迟更低
- SSH 兼容登录：仍通过 SSH 认证，再建立自己的 UDP 会话

## 典型使用场景

- 在移动办公、出差或弱网环境下远程连开发机
- 在高延迟跨地域连接里获得更顺滑的 shell 体验
- 配合 `tmux`、`screen` 等长期会话工具使用

## 为什么值得关注

- 它解决的是终端重度用户非常真实的交互痛点
- Mosh 多年持续被远程开发用户当作 SSH 替代增强层
- 对 CLI 生态来说，它属于“用起来就回不去”的基础体验类工具

## 类似项目
