# ImageMagick

## 一句话总结

一个支持 200 多种格式的图片处理套件，适合在终端里做转换、裁剪、合成和批量自动化处理。

## 它解决什么问题

- 图片批处理在 GUI 工具里重复劳动很多，不适合自动化
- 多格式转换、缩放、拼接和水印等任务需要更可组合的命令行接口
- 团队想把图片处理接进脚本、服务端或生成流水线时，需要稳定底座

## 适合谁

- 需要批量处理图片素材的开发者和设计相关工作者
- 想把图片转换和合成任务自动化的人
- 需要在服务器或脚本环境中做图像处理的团队

## 核心能力

- 格式广泛：支持 JPEG、PNG、GIF、TIFF、Ultra HDR 等大量格式
- 图片处理丰富：覆盖缩放、裁剪、旋转、合成、文字、水印和特效
- 自动化友好：适合构建复杂但稳定的图像处理流水线
- API + CLI 兼备：既能在终端里用，也能集成进程序

## 典型使用场景

- 批量图片格式转换和尺寸调整
- 添加水印、文字、边框或生成拼图
- PDF 转图片、图片转缩略图或站点资源预处理

## 为什么值得关注

- 它是命令行图像处理领域最经典的基础设施之一
- 能力足够深，适合从简单脚本一路扩展到复杂图像流水线
- 对 AI agent 来说，它同样适合“生成命令后执行”的使用方式

## 类似项目

- [FFmpeg](ffmpeg.md) - 同样是媒体处理基础设施，但核心对象是音视频
- [Pandoc](pandoc.md) - 同样强调格式转换，但对象是文档而不是图片
